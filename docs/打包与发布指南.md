## 一、一次性准备

### 1.1 Apple Developer ID Application 证书

**前提：** 已加入 Apple Developer Program（$99/年）。

**申请步骤：**

1. 浏览器打开 https://developer.apple.com/account/resources/certificates
2. 点 ➕ → 选 **Developer ID Application**
3. 在「钥匙串访问」生成 CSR：
   - 菜单 →「证书助理」→「从证书颁发机构请求证书」
   - 邮箱填 Apple Developer 账号邮箱
   - CA 邮箱留空
   - 选「存储到磁盘」
4. 上传生成的 `.certSigningRequest` 文件
5. 下载产生的 `.cer` → 双击装入钥匙串
6. 验证：

```bash
security find-identity -v -p codesigning
# 应出现一行：Developer ID Application: 你的姓名 (TEAM_ID)
```

> ⚠️ 这把证书是**分发证书**，跟你已有的 `Apple Development` 证书完全不同。开发证书签的包发给别人，Gatekeeper 会拦下；Developer ID 才能进入公网分发。

### 1.2 Apple 公证凭证（二选一）

发布证书签完包还要走 Apple 公证（Notarization）。两种凭证方式：

#### 方式 A：App Store Connect API Key（推荐）

**优点：** 永不过期、CI 友好。

**申请：**

1. 浏览器打开 https://appstoreconnect.apple.com/access/integrations/api
2. 点 ➕ Generate API Key
3. 角色至少选 **Developer**
4. 下载 `.p8` 文件（**只能下载一次**，丢了重申）
5. 记录三个值：
   - **Issuer ID**（页面顶部的 UUID）
   - **Key ID**（10 位字母数字）
   - `.p8` 文件路径

#### 方式 B：Apple ID + App 专用密码

**优点：** 上手快。**缺点：** 6 个月需轮换。

**申请：**

1. 浏览器打开 https://appleid.apple.com/account/manage
2. 「登录与安全」→「App 专用密码」→「生成」
3. 备注随便写（例如 "FotoForge notarization"）
4. 拷贝 4 段 4 位字母（形如 `xxxx-xxxx-xxxx-xxxx`）

> ⚠️ 千万**不要用 Apple ID 登录密码**，公证会失败。

### 1.3 Tauri Updater 密钥（已完成，但要备份）

**已完成事项：**

- 密钥对生成：`~/.tauri/fotoforge.key`（私钥）+ `~/.tauri/fotoforge.key.pub`（公钥）
- 公钥已编入 [src-tauri/tauri.conf.json](../src-tauri/tauri.conf.json) 的 `plugins.updater.pubkey`
- 私钥已写入 `.env.production.local` 的 `TAURI_SIGNING_PRIVATE_KEY`

**🔴 必做：离线备份私钥**

```bash
# 把这两个文件备份到至少两个独立介质：
ls -la ~/.tauri/fotoforge.key{,.pub}

# 推荐备份方式：
# 1. 加密 U 盘
# 2. 1Password / Bitwarden 安全笔记
# 3. 公司加密文件柜
```

**⚠️ 私钥丢失的后果：** 现有用户**永远收不到新更新**。新版本就算签了别的密钥，老 App 二进制里编入的 pubkey 也对不上，会拒绝任何更新包。详见后续工作 TODO 第 4 条「签名密钥轮换」。

### 1.4 阿里云 OSS bucket

**步骤：**

1. 登录 https://oss.console.aliyun.com
2. 创建 bucket：
   - **名称：** `fotoforge-updates`（或自定义；填进 `.env.production.local`）
   - **Region：** 华南1（深圳） `oss-cn-shenzhen`
   - **读写权限：** 私有（用 AccessKey 上传，公网通过 OSS 自带域名只读访问）
   - **服务端加密：** 默认即可
3. 进 bucket → **跨域设置（CORS）** → 新增规则：
   - 来源：`*`
   - 方法：`GET, HEAD`
   - 允许 headers：`*`
4. 创建 RAM 用户 + AccessKey：
   - https://ram.console.aliyun.com/users → 创建用户
   - 选「OpenAPI 调用访问」
   - 给该用户授权 `AliyunOSSFullAccess`（或更细的 bucket 级权限）
   - **保存 AccessKeyId 和 AccessKeySecret**（Secret 只显示一次）

### 1.5 绑定自定义域名

`ai520.wiki` 已备案，所以可以直接用：

1. OSS 控制台 → bucket → **域名管理** → 绑定自定义域名
2. 域名填 `static.ai520.wiki`
3. 选「自动添加 CNAME」（或手动到域名服务商加 CNAME 指向 bucket 默认域名）
4. **强制使用 HTTPS：** 勾选；OSS 会自动签发 Let's Encrypt 证书

**等待 5-10 分钟生效**。验证：

```bash
curl -I https://static.ai520.wiki
# 应看到 200 或 403（403 也对，说明域名通了，只是根路径无文件）
```

### 1.6 配置 `.env.production.local`

文件位置：[/.env.production.local](../.env.production.local)（已创建，被 `.gitignore` 忽略）

**填入实际值：**

```bash
# === 1) Apple 签名 ===
APPLE_SIGNING_IDENTITY="Developer ID Application: 你的姓名 (你的TEAM_ID)"

# === 2) Apple 公证（方式 A，推荐） ===
APPLE_API_ISSUER="..."                        # 第 1.2 节的 Issuer ID
APPLE_API_KEY="..."                            # 第 1.2 节的 Key ID
APPLE_API_KEY_PATH="/Users/你/keys/AuthKey_XXX.p8"

# 或方式 B：
# APPLE_ID="2787716172@qq.com"
# APPLE_PASSWORD="xxxx-xxxx-xxxx-xxxx"
# APPLE_TEAM_ID="W4T9X727L6"

# === 3) Tauri Updater（已填好） ===
TAURI_SIGNING_PRIVATE_KEY="dW50cnVzdGVk..."   # 已存
TAURI_SIGNING_PRIVATE_KEY_PASSWORD="你的密码"  # 你生成密钥时设置的

# === 4) 阿里云 OSS ===
ALIYUN_OSS_ACCESS_KEY_ID="LTAI..."             # 第 1.4 节的 AccessKeyId
ALIYUN_OSS_ACCESS_KEY_SECRET="..."             # 第 1.4 节的 Secret
ALIYUN_OSS_BUCKET="fotoforge-updates"
ALIYUN_OSS_REGION="oss-cn-shenzhen"
ALIYUN_OSS_DOMAIN="static.ai520.wiki"
ALIYUN_OSS_PATH_PREFIX="fotoforge"
```

## 二、日常发版流程

每次发新版照这个流程跑。

### 2.1 升版本号（修 1 个文件）

```bash
# 例如发 1.0.2
pnpm version 1.0.2 --no-git-tag-version
```

> 这条命令只改 `package.json` 的 `version` 字段，**不会** auto-commit、不会打 git tag。
>
> `tauri.conf.json` 和 `Cargo.toml` 的版本号会在下一步 `build:mac-arm` 触发 `prebuild:mac-arm` 钩子时**自动同步**。

### 2.2 加载凭证 + 打包

```bash
# 一行加载所有正式凭证
set -a; source .env.production.local; set +a

# 选其一打包：
pnpm build:mac-arm    # Apple Silicon 专用（最常用）
pnpm build:mac-x64    # Intel 专用
pnpm build:mac        # 通用（universal binary，体积大）
```

构建期间会自动完成：

1. `prebuild:mac-arm` → `pnpm version:sync` → 同步版本号到 `tauri.conf.json` + `Cargo.toml`
2. `tsc -b && vite build` → 编译前端
3. `cargo build --release` → 编译 Rust（首次 ~3 分钟，后续 < 30 秒）
4. 打包 `.app` → 用 `APPLE_SIGNING_IDENTITY` 签名
5. **公证**（如果 `APPLE_API_*` 或 `APPLE_ID/PASSWORD/TEAM_ID` 都填了）
6. 打包 `.app.tar.gz` → 用 `TAURI_SIGNING_PRIVATE_KEY` 做 minisign 签名 → 产出 `.app.tar.gz.sig`
7. 打包 `.dmg` → 用 `APPLE_SIGNING_IDENTITY` 签名

### 2.3 验证签名

```bash
# 设置产物路径变量（按你打的 target 调整）
APP=src-tauri/target/aarch64-apple-darwin/release/bundle/macos/FotoForge.app
DMG=src-tauri/target/aarch64-apple-darwin/release/bundle/dmg/FotoForge_*.dmg

# 1. .app 签名链
codesign -dv --verbose=4 $APP
# 关键字段：
# Authority=Developer ID Application: ... ← 必须是 Developer ID
# TeamIdentifier=W4T9X727L6
# Sealed Resources... (有这行 = 签名完整)

# 2. .dmg 签名
codesign -dv --verbose=4 $DMG | head

# 3. Gatekeeper 评估
spctl -a -vv $APP
# 通过则输出：
# accepted
# source=Notarized Developer ID

# 4. 公证状态（如果 stapler 装订成功）
stapler validate $APP
# The validate action worked!

# 5. minisign 签名
ls -la $APP.tar.gz.sig
# 文件存在 = updater 包就绪
```

**任意一步失败**都不能发版，回到「故障排除」。

### 2.4 上传到阿里云 OSS

```bash
# 凭证已 source 进当前 shell
pnpm publish:update
```

脚本自动完成：

- 收集 `*.app.tar.gz / *.sig / *.dmg`
- 上传到 `static.ai520.wiki/fotoforge/releases/<version>/`
- 生成 `latest.json`（含 minisign 签名 + 下载 URL）
- 上传 `latest.json` 覆盖到 `static.ai520.wiki/fotoforge/latest.json`
- `latest.json` 设置 `Cache-Control: no-cache`，CDN 不缓存
- 安装包设置 `Cache-Control: max-age=2592000`，缓存 30 天

成功输出末尾：

```
[publish] done. latest.json -> https://static.ai520.wiki/fotoforge/latest.json
```

### 2.5 端到端验证

```bash
# 1. latest.json 可访问且不缓存
curl -sI https://static.ai520.wiki/fotoforge/latest.json | grep -i "cache-control\|HTTP/"
# HTTP/2 200
# cache-control: no-cache, max-age=0

# 2. JSON 内容正确
curl -s https://static.ai520.wiki/fotoforge/latest.json | jq
# 应能看到 version、platforms、signature、url

# 3. 安装包可访问
curl -sI https://static.ai520.wiki/fotoforge/releases/1.0.2/FotoForge_1.0.2_aarch64.app.tar.gz | head -1
# HTTP/2 200
```

### 2.6 实测自动更新

最严谨的验证：用**老版本** App 测试能不能识别新版本。

```bash
# 1. 找一台/一个 Mac 还装着上一个版本（或备份的旧 .app）
# 2. 启动它
# 3. 等 3 秒，应该右上角飘出 toast：「发现新版本 1.0.2」
# 4. 点「下载并安装」→ 进度条 → 「下载完成」→ 点「立即重启」
# 5. 重启后的 App，关于页应显示新版本号
```

如果 Toast 没弹，看 [日志](#诊断：updater-没工作) 排查。

## 四、故障排除

### 4.1 DMG 打包失败：phantom mount

**报错：**

```
failed to bundle project error running bundle_dmg.sh
```

**原因：** 上一次构建中途死掉，留下了 `rw.NNNNN.FotoForge_*.dmg` 文件被挂载着。

**修复：**

```bash
# 找到 phantom mount
hdiutil info | grep -i fotoforge
# 例如 /dev/disk4

# 强制弹出
hdiutil detach /dev/disk4 -force

# 清空残留 bundle 目录
rm -rf src-tauri/target/aarch64-apple-darwin/release/bundle/

# 重新打包
pnpm build:mac-arm
```

### 4.2 公证失败：missing entitlements

**报错：**

```
Notary submission failed: ...
The signature does not include a secure timestamp...
```

**修复：** 检查 `APPLE_SIGNING_IDENTITY` 是 `Developer ID Application` 而**不是** `Apple Development`。开发证书签的包不能公证。

```bash
# 确认凭证
echo $APPLE_SIGNING_IDENTITY
# 应该是 "Developer ID Application: ..."
```

### 4.3 Updater 验签失败

**症状：** 客户端日志里出现 `signature verification failed`，下载完成但拒绝安装。

**原因：** `.app.tar.gz` 上传后被 OSS 透明压缩或修改，签名失效。

**修复：** 确认 OSS 没开启「自动压缩」选项；Cache-Control 设置正确。重新跑 `pnpm publish:update`。

### 4.4 OSS 上传 403 Forbidden

**症状：** `pnpm publish:update` 报 `403`。

**修复：**

1. 检查 RAM 用户授权，至少需要 `AliyunOSSFullAccess` 或针对该 bucket 的 `oss:PutObject` 权限
2. 检查 `.env.production.local` 的 AccessKey 是否填错
3. 检查 bucket 名字、region 是否对应

### 4.5 用户首次启动 Gatekeeper 警告

**症状：** 用户下载 DMG 双击安装，启动时弹「FotoForge 已损坏，无法打开」或「未识别的开发者」。

**原因：** 公证未通过 / stapler 未装订。

**修复：**

1. 本地跑 `stapler validate FotoForge.app` 确认装订状态
2. 如果未装订，单独跑：
   ```bash
   xcrun stapler staple src-tauri/target/aarch64-apple-darwin/release/bundle/macos/FotoForge.app
   ```
3. 用户侧临时方案：「系统设置 → 隐私与安全性」→ 看到 FotoForge 被拦截 → 点「仍要打开」

### 4.6 诊断：Updater 没工作

**症状：** 启动 3 秒后没有 toast。

**逐项排查：**

```bash
# 1. 客户端日志（启动后看）
tail -f ~/Library/Logs/com.fotoforge.app/*.log
# 关注 "updater" 关键字的行

# 2. 客户端能不能拿到 latest.json
curl https://static.ai520.wiki/fotoforge/latest.json
# 看 version 字段是否大于本机版本

# 3. 本机版本号
pnpm tauri info | grep version

# 4. 是否被 "跳过此版本" 设置抑制了
# 打开 App → 设置 → 更新 → 「已跳过的版本」 → 取消跳过

# 5. 确认 endpoint 配置正确
grep -A 2 endpoints src-tauri/tauri.conf.json
```

## 五、发版前检查清单

每次发版前照着勾。

**代码层：**

- [ ] 新功能/修复都已合入 `master` 分支
- [ ] `pnpm tsc --noEmit` 0 错误
- [ ] `cd src-tauri && cargo clippy -- -D warnings` 0 警告
- [ ] `cd src-tauri && cargo test` 全过
- [ ] CHANGELOG（如果有）已更新

**凭证层：**

- [ ] `.env.production.local` 内 6 个区段都填了真实值
- [ ] `~/.tauri/fotoforge.key` + `.pub` 已离线备份
- [ ] Apple Developer 账号未过期（每年 11 月续费）

**版本层：**

- [ ] `pnpm version <new>` 已执行
- [ ] git working directory 干净（除 `package.json` 这次的版本号改动）

**构建层：**

- [ ] `pnpm build:mac-arm`（或对应 target）成功
- [ ] `codesign -dv` 显示 Developer ID 而非 Apple Development
- [ ] `spctl -a -vv` 输出 `accepted source=Notarized Developer ID`
- [ ] `stapler validate` 通过
- [ ] `.app.tar.gz.sig` 存在且非空

**分发层：**

- [ ] `pnpm publish:update` 成功
- [ ] `curl https://static.ai520.wiki/fotoforge/latest.json` 返回新版本号
- [ ] `curl -I` 安装包返回 200
- [ ] 老版本能拉到新 latest.json 并提示更新

**通告层：**

- [ ] 官网下载链接已更新（如有）
- [ ] 用户群/社区已通知（如有）
